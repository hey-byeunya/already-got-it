# PRD: 이미 있어 (개인 재고 관리 웹앱)
**버전**: v1.3
**작성일**: 2026-07-22 (최종 수정: 2026-07-22)
**상태**: 1차 구현 및 배포 완료 (Vercel), UX 폴리시 + Claude Design 리디자인 반영

---

## 1. 개요

**한 줄 정의**: 기억력이 약하고 충동구매가 잦은 사람이, 사기 전에 "이미 있어?"를 스스로 확인하고 이미 가진 물건의 사용기한을 놓치지 않게 돕는 개인용 재고 관리 툴.
**배경**: 화장품을 시작점으로 하되, 특정 제품군에 종속되지 않는 범용 구조로 설계. 사용자가 겪는 문제는 제품군과 무관하게 반복되기 때문 (화장품, 식료품, 의류, 잡화 등으로 확장 가능해야 함).
**슬로건**: "물욕은 인정, 관리는 앱으로" — 이미 가진 걸 까먹고 또 사는 걸 막아주는, 아주 사적인 재고 관리 도구.
**문제 정의**
- 이미 가진 물건을 잊고 같은/비슷한 걸 또 산다 (기억력 문제)
- 사고 싶다는 충동이 들면 바로 결제로 이어진다 (충동구매)
- 사용기한이 있는 제품(특히 화장품)을 방치해 유통기한을 넘긴다 (관리 부재)
- 소유욕 자체가 과한 편이라, "가진 것"을 눈에 보이게 정리하는 것만으로도 자기 인식에 도움

## 2. 기능

- 로그인한 사용자가 자신의 물품(보유템, 화면 라벨은 "있템")과 위시리스트(화면 라벨은 "위시")를 관리하는 개인용 재고 관리를 한다.
- 보유템(이름·카테고리 자유 입력·수량·구매일·사용기한·메모·상태)을 등록·수정·삭제한다
- 이름·카테고리로 검색하고 사용기한 임박순으로 정렬해 D-day를 보여준다. 사용기한 임박(7일 이내)·지남·여유·기한 없음 4단계를 배지 색상과 아이콘으로 구분한다
- 상태(미개봉/사용중/다 씀)는 점(dot) + 색상 배지로, 보유 수량은 텍스트("N개") 대신 박스 아이콘 + 숫자로 표시한다
- 위시리스트에 이름·메모를 담아두고 '구매'를 누르면 보유템으로 옮긴다 (버튼 라벨은 "구매완료"에서 "구매"로 축약)
- 로그인 없이도 예시 데이터로 화면을 미리 체험할 수 있는 미리보기 화면(`/preview`, `/preview/wishlist`)을 제공한다
- 첫 로그인한 신규 사용자에게는 보유템·위시 예시를 1개씩 자동으로 채워줘서 빈 화면 대신 사용 방법을 바로 알 수 있게 한다
- 로그인 화면에 "이메일 저장" 체크박스를 두어, 다음 방문 시 이메일을 자동으로 채워준다 (비밀번호는 저장하지 않음)
- 로그인/보유템·위시 등록 폼에서는 Enter 키로 바로 제출된다 (메모 등 여러 줄 입력은 Enter가 줄바꿈으로 그대로 동작)
- 목록·폼 화면은 데이터를 불러오거나 제출을 처리하는 동안 화면 중앙에 로딩 스피너를 보여주고, 완료되면 부드럽게 전환된다
- 위시 항목을 구매 처리하면(성공이 확인된 뒤에만) 해당 카드가 자연스럽게 사라지는 애니메이션과 함께 목록에서 빠진다
- 보유템 등록/수정 화면의 수량 입력은 숫자 입력창 좌우에 −/+ 버튼이 있는 스테퍼로, 최소값 1 아래로는 내려가지 않는다
- 회원가입 시 닉네임(2~20자, 필수)을 입력받아 저장하고, 이용약관/개인정보 처리방침 동의 체크박스에 동의해야만 가입할 수 있다
- 로그인 화면에 "비밀번호를 잊으셨나요?" 링크를 두어, 이메일로 재설정 링크를 받아 새 비밀번호를 설정할 수 있다 (비밀번호 찾기 → 재설정 전체 플로우)
- 헤더의 로그아웃 버튼은 프로필 아바타 버튼으로 바뀌었고, 클릭하면 닉네임/이메일과 로그아웃 버튼이 담긴 드롭다운이 열린다
- 위시 항목에 참고 링크(선택)를 추가할 수 있고, 등록한 위시 항목을 눌러 이름·카테고리·참고 링크·메모를 수정할 수 있는 화면(위시 수정)이 있다
- 보유템/위시 등록·수정 폼과 목록의 카테고리 필터는 자유 텍스트 입력 대신, 기존에 사용한 카테고리를 칩으로 골라 선택하거나 "+ 새 카테고리"로 새로 추가하는 방식이다
- 보유템 상태(미개봉/사용중/다 씀)는 목록 카드에서는 진행형 세그먼트 바로, 등록/수정 폼에서는 3분할 세그먼트 버튼으로 선택한다
- 보유템/위시 목록이 완전히 비어 있을 때는 아이콘 + 안내 문구 + "첫 항목 추가하기" 버튼이 있는 빈 상태 화면을 보여준다 (검색/필터 결과가 없을 때는 기존의 단순 안내 문구만 표시)
- 전체 디자인은 Claude Design 목업을 기반으로 틸 그린(#00a090) 액센트 컬러와 Pretendard 폰트, 실제 로고 이미지로 리디자인되었다 (기존 반응형 웹 레이아웃 구조는 유지, 사용기한 임박도 배지 색상 구분은 그대로 유지)

## 3. 동작 (완성 기준)

- 로그인해야 보유템과 위시리스트에 접근할 수 있다
- 회원가입/로그인은 별도 화면(탭)으로 분리되어 있고, 회원가입 시에는 이메일 형식 검증과 비밀번호 확인(재입력 일치) 검증, 닉네임 길이 검증, 약관동의 필수 검증을 거친다
- 비밀번호 찾기 요청 시 입력한 이메일로 재설정 링크가 발송되고, 링크를 통해 접속한 화면에서만 새 비밀번호를 설정할 수 있다
- 등록·수정·삭제는 새로고침해도 남는다
- 위시리스트 항목을 구매 처리하면 위시리스트에서 사라지는 동시에 보유템에 추가되며, 둘 중 하나만 반영된 상태가 되지 않는다 (원자적 전환 — `mark_wishlist_purchased` 단일 RPC로 처리)
- 다른 계정으로 로그인하면 남의 물품은 보이지 않는다
- 로그인/회원가입 실패 시(이메일·비밀번호 오류, 이메일 미인증, 중복 가입, 이메일 발송 횟수 제한 등) 원인에 맞는 한국어 안내 문구를 보여준다
- 위시 구매 처리가 실패하면(오류로 인한 리다이렉트 등) 카드 사라짐 애니메이션은 실행되지 않고 기존 에러 안내만 표시된다 (성공 확인 전 낙관적 UI 변경 없음)

## 4. 데이터 모델

**보유템**
- 이름 (필수)
- 카테고리 (자유 텍스트, 필수)
- 수량 (필수, 기본값 1)
- 구매일 (필수, 등록 시 자동으로 오늘 날짜)
- 사용기한 (선택)
- 메모 (선택)
- 상태 (미개봉 / 사용중 / 다 씀)

**위시리스트**
- 이름 (필수)
- 카테고리 (선택)
- 참고 링크 (선택)
- 메모 (선택)
- 담은 날짜 (자동)

## 5. Must / Won't

**Must (이번에 포함)**
1. 로그인/회원가입 (이메일+비밀번호, Supabase Auth) — 로그인/회원가입 화면 분리, 이메일 형식·비밀번호 확인 검증, 실패 시 원인별 안내 문구
2. 보유템 등록/수정/삭제 (이름, 카테고리 자유 텍스트, 수량, 구매일 자동, 사용기한, 메모, 상태)
3. 물품 목록 보기 + 검색 (본인 물품만 보이게 — RLS 적용) + 사용기한 임박순 정렬/D-day 배지(단계별 색상·아이콘)
4. 위시리스트 등록/수정/삭제 + "구매 → 보유템 전환" (원자적 처리)
5. 로그인 없이 써볼 수 있는 예시 데이터 미리보기 화면
6. 신규 가입자용 예시 데이터 자동 채우기(온보딩)
7. UX 폴리시: 이메일 저장 체크박스, Enter 키 제출, 로딩 스피너, 위시 구매 성공 시 카드 사라짐 애니메이션, 수량 스테퍼
8. 회원가입 확장: 닉네임 입력, 약관동의, 비밀번호 찾기(재설정 메일 → 새 비밀번호 설정) 전체 플로우
9. 위시리스트 참고 링크 필드, 위시리스트 항목 수정 화면
10. 보유템/위시 카테고리 칩 선택 UI, 보유템 상태 세그먼트 스텝퍼 UI
11. Claude Design 목업 기반 리디자인 (틸 그린 액센트, Pretendard 폰트, 로고 에셋, 빈 목록 상태 화면)

**Won't (오늘은 일부러 뺄 것)**
1. 회원 정보 관리 (가입 후 닉네임/프로필 수정, 계정 탈퇴 등) — 비밀번호 재설정은 구현됨
2. 유통기한 알림/푸시 알림 (D-day는 화면에 표시만, 알림 없음)
3. 바코드 스캔·사진 인식 자동 등록
4. 카테고리 자동 분류/AI 추천
5. 소비 통계·차트 분석

## 6. 규칙 / 제약

- 무료 도구만 사용한다
- 비밀값(Supabase 키 등)은 코드에 노출하지 않는다 (`.env.local` + `.gitignore`). Service role key는 아예 사용하지 않고, 클라이언트에는 anon public key만 노출한다
- 배포는 **Vercel**(무료 Hobby 플랜)을 사용한다 — 이 앱은 Server Action·서버 세션 확인(proxy) 등 Next.js 서버 런타임에 의존하기 때문에 정적 호스팅(GitHub Pages 등)으로는 배포할 수 없다

## 7. 완성 기준 (done)

로그인해서 보유템을 등록하고, 새로고침해도 남아 있고, 이름/카테고리로 검색해서 확인할 수 있고, 위시리스트에 담았다가 보유템으로 전환할 수 있고, 다른 계정으로 로그인하면 내 물품이 보이지 않으면 성공. → 1차 구현 완료, Vercel에 배포하고 실제 계정으로 위 흐름을 확인함. 이 과정에서 발견된 `wishlist_items` RLS 정책 누락(구매 처리가 항상 실패하던 버그)도 수정 완료.

이후 UX 폴리시 라운드(이메일 저장, Enter 제출, 로딩 스피너, 구매 애니메이션, 수량 스테퍼)를 추가 반영함.

이후 Claude Design 목업 기반 리디자인 라운드(닉네임/약관동의/비밀번호 찾기, 위시 링크·수정 화면, 카테고리 칩 선택, 상태 세그먼트 스텝퍼, 틸 그린 컬러·Pretendard 폰트·로고, 빈 목록 상태 화면)를 추가 반영함.
