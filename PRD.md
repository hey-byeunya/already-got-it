# PRD: 이미 있어 (개인 재고 관리 웹앱)
**버전**: v1.0
**작성일**: 2026-07-22
**상태**: 초안 / 리뷰 전

---

## 1. 개요

**한 줄 정의**: 기억력이 약하고 충동구매가 잦은 사람이, 사기 전에 "이미 있어?"를 스스로 확인하고 이미 가진 물건의 사용기한을 놓치지 않게 돕는 개인용 재고 관리 툴.
**배경**: 화장품을 시작점으로 하되, 특정 제품군에 종속되지 않는 범용 구조로 설계. 사용자가 겪는 문제는 제품군과 무관하게 반복되기 때문 (화장품, 식료품, 의류, 잡화 등으로 확장 가능해야 함).
**문제 정의**
- 이미 가진 물건을 잊고 같은/비슷한 걸 또 산다 (기억력 문제)
- 사고 싶다는 충동이 들면 바로 결제로 이어진다 (충동구매)
- 사용기한이 있는 제품(특히 화장품)을 방치해 유통기한을 넘긴다 (관리 부재)
- 소유욕 자체가 과한 편이라, "가진 것"을 눈에 보이게 정리하는 것만으로도 자기 인식에 도움

## 2. 기능

- 로그인한 사용자가 자신의 물품(보유템)과 위시리스트를 관리하는 개인용 재고 관리를 한다.
- 보유템(이름·카테고리 자유 입력·수량·구매일·사용기한·메모·상태)을 등록·수정·삭제한다
- 이름·카테고리로 검색하고 사용기한 임박순으로 정렬해 D-day를 보여준다
- 위시리스트에 이름·메모를 담아두고 '구매완료'를 누르면 보유템으로 옮긴다

## 3. 동작 (완성 기준)

- 로그인해야 보유템과 위시리스트에 접근할 수 있다
- 등록·수정·삭제는 새로고침해도 남는다
- 위시리스트 항목을 구매완료 처리하면 위시리스트에서 사라지는 동시에 보유템에 추가되며, 둘 중 하나만 반영된 상태가 되지 않는다 (원자적 전환)
- 다른 계정으로 로그인하면 남의 물품은 보이지 않는다

## 4. 데이터 모델

**보유템**
- 이름 (필수)
- 카테고리 (자유 텍스트, 필수)
- 수량 (필수, 기본값 1)
- 구매일 (필수, 등록 시 자동으로 오늘 날짜)
- 사용기한 (선택)
- 메모 (선택)
- 상태 (미개봉 / 사용중 / 다 씀)

**위시리스트**
- 이름 (필수)
- 카테고리 (선택)
- 메모 (선택)
- 담은 날짜 (자동)

## 5. Must / Won't

**Must (이번에 포함)**
1. 로그인/회원가입 (이메일+비밀번호, Supabase Auth)
2. 보유템 등록/수정/삭제 (이름, 카테고리 자유 텍스트, 수량, 구매일 자동, 사용기한, 메모, 상태)
3. 물품 목록 보기 + 검색 (본인 물품만 보이게 — RLS 적용) + 사용기한 임박순 정렬/D-day 배지
4. 위시리스트 등록/삭제 + "구매완료 → 보유템 전환" (원자적 처리)

**Won't (오늘은 일부러 뺄 것)**
1. 회원 정보 관리 (닉네임/프로필 수정, 비밀번호 변경, 계정 탈퇴 등)
2. 유통기한 알림/푸시 알림 (D-day는 화면에 표시만, 알림 없음)
3. 바코드 스캔·사진 인식 자동 등록
4. 카테고리 자동 분류/AI 추천
5. 소비 통계·차트 분석

## 6. 규칙 / 제약

- 무료 도구만 사용한다
- 비밀값(Supabase 키 등)은 코드에 노출하지 않는다 (`.env.local` + `.gitignore`)

## 7. 완성 기준 (done)

로그인해서 보유템을 등록하고, 새로고침해도 남아 있고, 이름/카테고리로 검색해서 확인할 수 있고, 위시리스트에 담았다가 보유템으로 전환할 수 있고, 다른 계정으로 로그인하면 내 물품이 보이지 않으면 오늘 성공.
